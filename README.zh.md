# Obsidian Signature 插件

> ⚠️ **AI 生成项目** — 本插件完全由 AI 助手（Anthropic 的 Claude）构建，仅在人工方向和需求的引导下完成。没有任何代码是手动编写的。概念、功能决策和测试均由人类作者提供。

在笔记中内联签名，包含姓名、职位、时间戳和唯一哈希值——用于内部控制和可追溯性，无官方法律效力。

---

## 功能

- **多语言触发标签** — 每种语言有专属标签：
  - 中文：`[签名]` → `[已签名: 姓名 - 职位 | 2026-06-12 14:32 | a1b2c3d4]`
  - 英文：`[signature]` → `[SIGNED: Name - Role | 2026-06-12 14:32 | a1b2c3d4]`
  - 葡萄牙语：`[assinatura]` → `[ASSINADO: Nome - Cargo | 2026-06-12 14:32 | a1b2c3d4]`
- **所有标签均可通用检测** — 无论当前语言设置如何，任意标签均可使用
- **按签署人设置密码** — 可选；以 FNV-1a 哈希存储（不可恢复）
- **签名受保护** — CM6 事务过滤器阻止直接编辑 `[已签名: ...]` 文本块
- **删除签名命令** — `Ctrl+P` → *删除光标处的签名* — 如已设置密码则需验证
- **SMTP 邮件通知** — 可选；每次应用签名后自动发送
- **三语言界面** — English、Português (pt-BR)、中文

---

## 安装（手动）

1. 下载或构建 `main.js`、`manifest.json` 和 `styles.css`
2. 复制到 `<您的库>/.obsidian/plugins/obsidian-signature/`
3. 在 Obsidian 中：**设置 → 社区插件 → 关闭安全模式 → 启用 Signature**

### 从源码构建

```bash
npm install --legacy-peer-deps
npm run build
```

---

## 使用方法

1. 在笔记任意位置写入 `[签名]`
2. 在编辑器中点击高亮文本
3. 如果配置了多个签署人，从列表中选择
4. 如果签署人设置了密码，请确认（最多 3 次尝试）
5. 标签将被替换为完整的签名块

**删除**签名：将光标置于签名行 → `Ctrl+P` → **删除光标处的签名**

---

## 设置

打开 **设置 → Signature**：

| 部分 | 功能 |
|------|------|
| 语言 | 切换界面语言：English、pt-BR、中文 |
| 签署人 | 为每个签署人添加姓名、职位和可选密码 |
| 邮件通知 | 配置 SMTP 服务器和收件人列表 |

---

## 邮件通知

- 支持任意 SMTP 服务器（Gmail、Outlook、自定义）
- Gmail 用户：请使用**应用专用密码** — 前往 `myaccount.google.com → 安全 → 应用专用密码`
- 启用前请点击**发送测试邮件**按钮验证连接

> ⚠️ SMTP 密码以明文存储在 `data.json` 中。**请使用应用专用密码，切勿使用主账户密码。** `data.json` 默认已在 `.gitignore` 中。

---

## 安全说明

- 签署人密码以 **FNV-1a 32 位哈希**存储——不可逆，不可恢复
- 签名哈希（`a1b2c3d4`）由姓名 + 职位 + 时间戳派生——是唯一性标识符，非密码学证明
- 本插件提供**内部控制**，不具备法律效力
