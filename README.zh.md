# Obsidian Signature 插件

> ⚠️ **AI 生成项目** — 本插件完全由 AI 助手（Anthropic 的 Claude）构建，仅在人工方向和需求的引导下完成。没有任何代码是手动编写的。概念、功能决策和测试均由人类作者提供。

在笔记中内联签名，配有防篡改哈希值。精确检测篡改——指出哪个签名失败及其原因。

---

## 为什么创建此插件

我们团队正处于文档标准化阶段。作为这一过程的一部分，我们需要一个可靠的方式来追踪**谁做了什么、是否经过领导审核、以及在必要时谁重构了文档**。

我们使用 Obsidian 作为共享知识库，因为它天然映射到共享服务器上的一个文件夹——团队中的每个人都可以无障碍地读写笔记，并且与 AI 辅助工作流配合良好。

问题在于可追溯性：Markdown 文件能告诉你*改了什么*，但无法说明*谁批准了*或*是否经过审核*。传统解决方案（PDF 导出、带修订记录的 Word、外部签名工具）对于快节奏的文档工作流来说过于繁重。

本插件通过在笔记中直接嵌入轻量级内联签名来解决这个问题。每个签名记录签署人的身份、时间戳以及签署时文档内容的哈希值。如果文档在签署后被修改，哈希值将不再匹配——使任何签署后的更改立即可见。

**这些签名不具有法律效力。** 它们的价值是实用性的：直接在文档内部创建轻量级审计追踪，让人一目了然地知道谁接触了文档、担任什么角色，以及自签署以来内容是否发生了变化。

## 工作原理

### 签署笔记

在文档任意位置输入占位符标签：

| 语言 | 标签 |
|------|------|
| 葡萄牙语 | `[assinatura]` |
| 英语 | `[signature]` |
| 中文 | `[签名]` |

点击标签。如果配置了多个签署人，会显示选择器。标签被替换为如下块：

```
[已签名: 张三 - 经理 | 2026-06-15 14:32 | a3f2b1c4.e8d9f7a2]
```

该块包含两个 FNV-1a 32-bit 哈希值：
- **idHash**（`a3f2b1c4`）— 身份哈希：`fnv32a(姓名 + "|" + 职位 + "|" + 时间戳)`
- **contentHash**（`e8d9f7a2`）— 内容哈希：`fnv32a(去除所有签名后的文档文本)`

签署后对文档的任何更改都会使 `contentHash` 失效。

### 锁定模式

如果签署人启用了**"签名时锁定文档"**，块将使用 `[已锁定: ...]` 前缀。包含任何有效锁定签名的文档将完全变为只读，直到签名被移除。

## 命令

| 命令 | 功能 |
|------|------|
| `验证当前笔记中的签名` | 检查所有签名并报告**哪个**签名失败及**原因**（身份哈希？内容哈希？两者？） |
| `删除光标处的签名` | 删除光标下的签名。如果签署人设有密码则提示输入。 |
| `打开签名仪表板` | 打开侧边栏，列出整个 vault 中的所有签名及其状态 |
| `通过哈希验证签名` | 打开模态框，手动将哈希对与粘贴的内容进行验证 |

## 签名仪表板

点击工具栏中的 🖊 图标或使用**"打开签名仪表板"**命令打开侧边栏。

该面板扫描 vault 中的所有 Markdown 文件，并列出每个签名的以下信息：

| 列 | 内容 |
|----|------|
| 图标 | ✅ 有效 · ❌ 已篡改 · ⚠️ 旧版 · 🔒 已锁定 |
| 姓名与职位 | 签署人信息 |
| 日期 | 签名时间戳 |
| 状态 | 文字说明 |

**点击任意行** → 打开对应笔记并显示详情模态框，包含所有签名信息以及**哈希对比**（存储值 vs 当前值），确认内容是否在签署后被修改。

顶部的**状态过滤器**可仅查看已篡改、旧版等签名。

**↺ 按钮**刷新 vault 扫描。

## 篡改报告

检测到篡改时，通知会精确显示每个签名的问题：

```
❌ 检测到 2 个被篡改的签名！
  • 张三（经理）：内容哈希不匹配
  • 李四（总监）：两个哈希均不匹配
```

可能的失败类型：
- **身份哈希不匹配** — 块内的姓名/职位/时间戳被修改
- **内容哈希不匹配** — 文档正文在签署后被更改
- **两个哈希均不匹配** — 块和内容均被修改

---

## 哈希验证

使用**"通过哈希验证签名"**在原始笔记上下文之外验证签名。

**输入：**
1. 哈希对：`xxxxxxxx.xxxxxxxx`（直接从 `[已签名: ...]` 块中复制）
2. 文档内容**不含**签名（粘贴纯文本）

插件重新计算 `fnv32a(stripSignatures(内容))` 并与存储的 `contentHash` 进行比较。

**CLI 替代方案** — 无需打开 Obsidian，使用 `verify.mjs`：
```bash
node verify.mjs 笔记.md
node verify.mjs --dir ./vault
node verify.mjs --hash a3f2b1c4.e8d9f7a2 笔记.md
```

## 设置

设置分为两个选项卡：

### ✍️ 签署人选项卡

每个签署人都有一个可折叠卡片（默认关闭）。点击展开：

| 字段 | 描述 |
|------|------|
| 姓名 | 显示在签名块中 |
| 职位 | 显示在签名块中 |
| 密码（可选） | 设置后，移除签名时需要输入 |
| **管理员** | 访问仪表板并有权移除任何签名 |
| **签名时锁定** | 使用 LOCKED 前缀并使文档变为只读 |

密码管理通过独立模态框：设置 / 更改（需要当前密码）/ 移除（需要当前密码）。

### ⚙️ 管理器选项卡

- **语言** — pt-BR / English / 中文
- **打开时验证** — 打开任意笔记时自动检查签名
- **邮件通知** — SMTP 配置，支持单独添加/删除收件人，以及篡改警报开关

## 签名块格式

```
[已签名: 姓名 - 职位 | YYYY-MM-DD HH:MM | idHash.contentHash]
[已锁定: 姓名 - 职位 | YYYY-MM-DD HH:MM | idHash.contentHash]
```

**向后兼容：** 旧格式（无 `contentHash`）的块被识别为 `⚠️ 旧版`——正常显示但无法进行完整性验证。

---

## 哈希算法

```
fnv32a(str):
  hash = 2166136261       // FNV 偏移基准
  对每个字符:
    hash ^= charCode
    hash *= 16777619      // FNV 质数
    hash &= 0xFFFFFFFF
  返回 hash.toString(16).padStart(8, '0')

idHash      = fnv32a(姓名 + "|" + 职位 + "|" + 时间戳)
contentHash = fnv32a(去除所有签名块和占位符后的文档)
```

> FNV-1a 32-bit 快速且足以用于非正式篡改检测。如需法律或安全关键用途，请考虑 SHA-256。
